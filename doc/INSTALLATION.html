Admin Panel Installation Guide

Complete setup guide for the Next.js Admin Dashboard Template. This is a frontend-only application with mock data support.

1. Project Overview

Admin Panel is a modern, responsive dashboard template built with Next.js 16, TypeScript, and Tailwind CSS. It provides a complete admin interface with multiple dashboard variants, data visualization charts, customer management, order tracking, and more.

Quick Stats:
  • Latest Next.js 16 with App Router
  • TypeScript for type safety
  • Tailwind CSS for styling
  • React Query for data fetching

5. Installation Steps

Step 1: Navigate to Project Directory

cd PANEL

Step 2: Install Dependencies

Install all required npm packages:

npm install

This will install all dependencies defined in package.json.

Step 3: Environment Configuration

Create a local environment file by copying the example:

Windows PowerShell:

Copy-Item .env.example .env.local

macOS/Linux/Git Bash:

cp .env.example .env.local

Step 4: Configure Environment Variables

Edit .env.local with your preferred settings:

# Application Name
NEXT_PUBLIC_APP_NAME=Admin Panel

# API Base URL (for your backend)
NEXT_PUBLIC_API_BASE_URL=http://localhost:4000/api

# Optional: Enable debug mode
NEXT_PUBLIC_DEBUG=false
Tip: The application includes mock API calls in lib/api.ts. For development, you can use the mock data without setting up a backend server.

Step 5: Run Development Server

Start the development server:

npm run dev

Open your browser and navigate to:

http://localhost:3000

Environment Variables for Production

Set these environment variables on your production server:

NEXT_PUBLIC_APP_NAME=Admin Panel
NEXT_PUBLIC_API_BASE_URL=https://your-api-domain.com/api
NEXT_PUBLIC_DEBUG=false
Note: Always set NEXT_PUBLIC_DEBUG=false in production to disable debug features.

9. Authentication

This template includes a mocked authentication system for demonstration purposes. The app checks for a token stored in localStorage.

How Authentication Works:
  • On first load, if no token exists, user is redirected to /lock
  • Login page saves a mock token to localStorage.admin_token
  • After login, user is redirected to /dashboard
  • Token presence is checked via AuthGuard component

To Implement Real Authentication:

  1. Update src/lib/api.ts with real API calls
  2. Modify the login function in src/app/lock/page.tsx
  3. Update src/components/layout/AuthGuard.tsx to validate real tokens
  4. Add JWT/session validation logic

10. API Configuration

The application uses React Query for data fetching. All API functions are defined in src/lib/api.ts.

Mock Data

By default, the app uses mock data functions. These return simulated data for development without a backend.

Connecting to a Real API

  1. Update NEXT_PUBLIC_API_BASE_URL in your environment file
  2. Modify the API functions in src/lib/api.ts
  3. Replace mock functions with real fetch/axios calls
// Example: Replace mock function
// Before (mock):
async function getDashboardStats() {
  return mockDashboardStats;
}

// After (real API):
async function getDashboardStats() {
  const response = await fetch(`${process.env.NEXT_PUBLIC_API_BASE_URL}/stats`);
  return response.json();
}

12. Troubleshooting

Common Issues

Issue Solution
Port 3000 in use Run npm run dev -- -p 3001 or kill the process
TypeScript errors Run npm install to ensure all types are installed
Build fails Check for lint errors with npm run lint
Styles not loading Ensure globals.css imports Tailwind directives
Charts not rendering Check browser console for D3/Recharts errors
Still having issues?
  • Clear node_modules and reinstall: rm -rf node_modules && npm install
  • Clear Next.js cache: rm -rf .next
  • Check Node.js version compatibility
